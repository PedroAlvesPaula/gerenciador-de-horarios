Este é o repositório principal de um sistema de gestão de horários. O projeto é uma aplicação Full-Stack dividida em dois ambientes principais: a interface de usuário (Front-end) e a API de serviços (Back-end).

## 🗂️ Estrutura do Repositório

O projeto foi pensado para manter a separação clara de responsabilidades entre o lado do cliente e o lado do servidor:

```text
PHBarber/
 ├── backend/    # API e regras de negócio (Em desenvolvimento)
 ├── frontend/   # Interface do usuário (Landing Page e Painel)
 └── README.md   # Documentação principal
```

## Front-end (Interface do Usuário)

O Front-end da aplicação já está estruturado com uma arquitetura modular. Ele engloba a **Landing Page**, **Painel de Autenticação/Agendamento (em desenvolvimento)** e **Página de Administração (em desenvolvimento)**.

- **Tecnologias principais:** React, TypeScript, Vite, Material UI (v6) e React Router DOM.
- **Status:** Arquitetura base, Landing Page e fluxo de Login concluídos (utilizando dados mockados temporariamente).
